# Генератор посещаемости на неделю

Создаёт тестовый `attendance.json` на 7 дней (учебная неделя).

- **Контингент** — из `students.json` (отделения, группы, студенты) не меняются.
- **Пропуски** — либо по распределению из эталонного attendance (`-ref`), либо по встроенному «реалистичному» распределению (не всё зелёное).

## Запуск

**Неделя по эталону (15.01, 18.02 и т.д.) — «слить» и размазать на неделю:**

```bash
cd backendupdate/backend
go run ./cmd/generate-week-attendance -out ../public -week 2026-02-16 -ref ../public/attendance.json
```

Сначала сохрани текущий `attendance.json` (с реальными 15.01 и др.) в отдельный файл, например `attendance_ref.json`. Потом запусти генератор с `-ref ../public/attendance_ref.json`, а `-out ../public` перезапишет `attendance.json` уже недельными данными с тем же распределением пропусков.

**Без эталона (моковая неделя со встроенным распределением):**

```bash
go run ./cmd/generate-week-attendance -out ../public -week 2026-02-16
```

Параметры:

- `-out` — каталог, куда писать `attendance.json` и откуда читать `students.json`.
- `-week` — понедельник недели YYYY-MM-DD (по умолчанию 2026-02-16).
- `-ref` — путь к эталонному `attendance.json` (данные за 15.01, 18.02 и т.д.). По нему считается распределение значений пропусков (0, 2, 4, 6, 8), и по этому распределению генерируются пропуски на 7 дней для всего контингента из `students.json`.

## Расписание (schedule.json)

Неделя в расписании задаётся полем `period` (например 16.02.2026–22.02.2026). Задай `-week 2026-02-16`, чтобы даты посещаемости совпадали с этой неделей.
