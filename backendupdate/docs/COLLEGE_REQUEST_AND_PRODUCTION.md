# Что запрашивать у колледжа и подготовка к продакшену

## 1. Данные для автоматической работы системы

Ниже — список того, что нужно получать от колледжа **регулярно** (ежедневно/еженедельно) в согласованном формате.

---

### 📅 Дата
- Рабочая дата (день занятий) в формате **YYYY-MM-DD** или **DD.MM.YYYY** (единый формат во всех выгрузках).

---

### 👥 Группа
- Код группы (например `1ис1`, `2д1`).
- Название отделения (например «Отделение программирования»).

---

### 📚 Предмет (дисциплина)
- Полное название дисциплины (как в учебном плане).
- При необходимости — код дисциплины для связки с оценками.

---

### 👨‍🏫 Преподаватель
- ФИО в едином виде (например «Фамилия И.О.»).

---

### 🕒 Пара (номер занятия)
- Номер пары в учебном дне: **1–6** (или 1–8, если в колледже больше пар).
- Время начала/окончания пары (опционально, для «текущей пары» на дашборде).

---

### 📍 Аудитория
- Корпус и номер аудитории (например «Основной корпус-Аудитория 105») — для отображения в расписании.

---

### ✅ Присутствие (ФИО + статус)
- **ФИО студента** (полное, как в приказе).
- **Статус по паре:** присутствовал / отсутствовал (или количество пропущенных часов/пар за день).
- **Дата** — к какой дате относится отметка.

Формат выгрузки: по дням и парам (кто был на паре) или свод по дням (пропуски за день в часах/парах).

---

### 📝 Оценка
- Оценка за занятие/контрольную (если колледж ведёт учёт и готов выгружать).
- Поля: дата, группа, дисциплина, студент (ФИО), оценка.
ghjghj
*(Сейчас в системе оценок нет — при появлении выгрузки их можно добавить в схему и отображение.)*

---

### 🔁 Замена (если есть)
- Признак замены (другая дисциплина / другой преподаватель).
- Поля: дата, пара, группа, фактическая дисциплина, фактический преподаватель.

*(Сейчас отдельного поля замены нет — при необходимости можно добавить в структуру расписания.)*

---

### 📊 Итог за день
- Опционально: свод по каждому студенту за день (сколько пар посетил, сколько пропустил).
- Можно считать на стороне системы из детализации по парам.

---

### 📈 Итог за неделю
- Опционально: свод за неделю по студенту/группе.
- Аналогично — можно считать из ежедневных данных.

---

### 📂 Формат выгрузки
- **Предпочтительно:** Excel (`.xls` или `.xlsx`) с фиксированными именами листов и колонок (см. ниже).
- Допустимо: CSV с разделителем и кодировкой (например UTF-8), если конвертеры доработать под CSV.

---

## 2. Технический список полей для регулярной выгрузки

| № | Поле | Обязательность | Пример |
|---|------|----------------|--------|
| 1 | Дата | да | 2026-02-20 |
| 2 | Отделение | да | Отделение программирования |
| 3 | Группа | да | 1ис1 |
| 4 | ФИО студента | да | Иванов И.И. |
| 5 | Номер пары | да | 1–6 |
| 6 | Дисциплина | да | Математика |
| 7 | Преподаватель | да | Петрова П.П. |
| 8 | Аудитория | нет | Аудитория 105 |
| 9 | Присутствие (да/нет или часы пропусков) | да | да / нет или 0 / 2 / 4 … |
| 10 | Оценка | нет | 5 |
| 11 | Признак замены | нет | да / нет |

Минимальный набор для текущей системы: **1–9**. Остальное — для расширения (оценки, замены, итоги).

## 3. Итог по проекту и шаги к продакшену

### Сделано
- Схема данных описана (`docs/DATA_SCHEMA.md`): статичные/динамичные поля, связи между JSON.
- Генератор тестовых данных на 7 дней: `cmd/generate-week-attendance` (посещаемость из `students.json`, даты и пропуски генерируются).
- Фронт использует только данные с API (моковые подстановки только как fallback при отсутствии данных).
- Оперативный режим — только «сегодня»; исторический — с выбором даты; таблицы по парам (6 пар) и по отделениям/группам на реальных данных.
- Документация по Excel и развёртыванию (`docs/EXCEL_AND_DEPLOY.md`), список запросов к колледжу (этот файл).

### Что сделать для продакшена
1. **Данные** — получать от колледжа выгрузки по согласованному формату (см. список полей выше).
2. **Конвертеры** — проверить и при необходимости доработать под актуальные шаблоны Excel колледжа (имена листов, колонок, форматы дат).
3. **Бэкапы** — перед каждой перезаписью JSON (или БД) делать копию текущих файлов.
4. **Окружение** — все пути и секреты в переменных окружения; логи и мониторинг ошибок.
5. **Права** — загрузка Excel и «обновить данные» только у роли admin.
6. **При росте нагрузки** — рассмотреть переход с JSON на БД (SQLite или PostgreSQL) и вынести конвертеры в фоновые задачи.

После выполнения этих шагов систему можно считать готовой к промышленной эксплуатации при условии регулярных выгрузок от колледжа.
