## Оптимизация и порядок в проекте

### Бэкенд
- **Сервисы:** разделение по зонам ответственности (schedule, attendance, reconciliation, dashboard).
- **Дедупликация:** в `schedule` и reconciliation студент не дублируется по нескольким парам в один день.
- **Кэш:** сверка по дате/паре кэшируется в памяти с TTL 5 мин.
- **Конвертеры:** вынесены в `internal/converter`; CLI в `cmd/` (convert-schedule, convert-statement, convert-master, generate-week-attendance).

### Фронт
- **Реальные данные:** таблицы по парам, отделениям и группам строятся из API (reconcile по дате и паре).
- **Fallback:** при отсутствии данных по парам используется агрегат за день (`drillToAttendanceRows`) только для отображения, без подмены источника истины.
- **Режимы:** оперативный (только сегодня, без выбора даты); исторический (выбор даты, 6 пар в таблице).

### Данные
- **Демо на неделю:** генератор `cmd/generate-week-attendance` создаёт `attendance.json` на 7 дней по контингенту из `students.json`.
- **Расписание:** `schedule.json` уже содержит записи на неделю (period 16.02–22.02); даты в записях приведены к единому формату на бэкенде.

## Рабочая демо-версия с недельными данными

1. Сгенерировать посещаемость на неделю:
   ```bash
   cd backendupdate/backend
   go run ./cmd/generate-week-attendance -out ../public -week 2026-02-16
   ```
2. Запустить бэкенд и фронт.
3. На главной и в мониторинге отображаются данные за «сегодня»; в историческом дашборде можно выбрать любую дату из этой недели и видеть 6 пар с реальными цифрами.

## Что сделать для продакшена

- Получать от колледжа выгрузки по формату из [COLLEGE_REQUEST_AND_PRODUCTION.md](./COLLEGE_REQUEST_AND_PRODUCTION.md).
- Настроить конвертеры под актуальные Excel-шаблоны колледжа.
- Включить бэкапы JSON перед перезаписью.
- Вынести конфигурацию и секреты в env.
- Ограничить загрузку Excel и обновление данных ролью admin.
- При росте нагрузки рассмотреть переход на БД (см. [EXCEL_AND_DEPLOY.md](./EXCEL_AND_DEPLOY.md)).
